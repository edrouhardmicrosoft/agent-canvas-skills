## Violations

### Issue #1: Link Missing Accessible Name

| Property | Value |
|----------|-------|
| **Rule ID** | `link-name` |
| **Severity** | Serious |
| **WCAG** | 2.4.4 (Level A), 4.1.2 (Level A) |
| **Section 508** | 22.a |

**Element:**
```html
<a class="framer-vabebt framer-xcdope" 
   data-framer-name="LinkedIn" 
   href="https://www.linkedin.com/in/erikdrouhard" 
   target="_blank" 
   rel="noopener">
```

**CSS Selector:** `.framer-vabebt`

**Problem:**  
The LinkedIn link has no accessible text. Screen reader users cannot determine the link's purpose.

**Why it fails:**
- Element does not have text visible to screen readers
- `aria-label` attribute does not exist or is empty
- `aria-labelledby` attribute does not exist
- Element has no `title` attribute
- Element is in tab order but has no accessible text

**Fix Options (choose one):**

1. **Add `aria-label`** (Recommended):
   ```html
   <a class="framer-vabebt framer-xcdope" 
      aria-label="Erik Drouhard on LinkedIn"
      href="https://www.linkedin.com/in/erikdrouhard" 
      target="_blank" 
      rel="noopener">
   ```

2. **Add visible text**:
   ```html
   <a class="framer-vabebt framer-xcdope" 
      href="https://www.linkedin.com/in/erikdrouhard" 
      target="_blank" 
      rel="noopener">
      LinkedIn
   </a>
   ```

3. **Add `title` attribute** (Less preferred):
   ```html
   <a class="framer-vabebt framer-xcdope" 
      title="Visit Erik's LinkedIn profile"
      href="https://www.linkedin.com/in/erikdrouhard" 
      target="_blank" 
      rel="noopener">
   ```

**Reference:** [Deque University - link-name](https://dequeuniversity.com/rules/axe/4.11/link-name?application=axeAPI)
